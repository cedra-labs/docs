Cedra Fullnode Overview

A Cedra node is an always‚Äëon service that tracks and serves the canonical state of the Cedra blockchain. All wallets, explorers, exchanges, and dapps communicate with the chain through these nodes.

Cedra‚Äëcore can operate in two mutually‚Äëexclusive roles:

RolePurpose
Validator nodeExecutes and signs blocks, participates in BFT consensus, and earns staking rewards.
FullnodeReplicates the ledger for read/write access but does not vote in consensus.

Internal Components‚Äã

A Cedra node is composed of several logical services, each running inside the same binary:

  1. REST API service ‚Äì HTTP+JSON entry point for clients.
  2. Mempool ‚Äì Admits, orders, and gossips pending transactions.
  3. Execution ‚Äì Runs transactions through MoveVM and produces deterministic outputs.
  4. MoveVM ‚Äì Executes Move bytecode with linear‚Äëtype resource safety.
  5. Storage ‚Äì Persists blocks, state, and events in a RocksDB‚Äëbacked datastore.
  6. State synchronizer ‚Äì Streams certified ledger data from upstream peers and keeps local state in lock‚Äëstep.

Tip¬†: The same cedra‚Äëcore binary can be started with --role validator or --role fullnode to switch behavior.

Fullnode Operation‚Äã

Fullnodes may be run by anyone. They bootstrap by downloading the ledger from an upstream peer and then either:

  • Re‚Äëexecuting every historical transaction to independently verify state, or
  • Replaying outputs (faster) after checking the epoch accumulator root signed by the validators.

Fullnodes also accept client‚Äësubmitted transactions and forward them to validators, but they do not take part in consensus voting.

Fullnode typeUpstream peer
Validator fullnodeDirectly follows a validator node.
Public fullnodeFollows another fullnode.

The functionality is identical - the label only reflects who the upstream is.

Why Run a Fullnode?‚Äã

  • Use the local REST interface for unrestricted blockchain interaction.
  • Obtain a consistent, trust‚Äëbut‚Äëverify view of the Cedra ledger.
  • Bypass public endpoint rate limits for high‚Äëvolume reads.
  • Run custom analytics over historical data and events.
  • Receive real‚Äëtime on‚Äëchain notifications without third‚Äëparty services.
